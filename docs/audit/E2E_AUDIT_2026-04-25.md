**SEC-01 (HIGH): Auto-registration of arbitrary tokens**
Any Bearer token sent to the API is automatically registered as a free-tier key via `ensure_api_key()` in database.py. This means:
- Rate limits can be bypassed by rotating tokens
- The 500 calls/month limit is per-key, not per-entity
- No real authentication barrier — any string becomes a valid key
- **Mitigation:** Add email verification before key activation, or require signup flow

### 2.2 Authorization & Multi-Tenancy

|------|--------|----------|

| Plan-gated tools | PASS | `require_plan()` checks DB plan column |

**SEC-02 (HIGH): MCP annotations missing requiresAuth**
All 62 tools return `requiresAuth: undefined` in MCP annotations. Auth enforcement happens at handler level via `require_auth()`, but clients have no way to know which tools need auth before calling them. Fix: Add `annotations=ToolAnnotations(requiresAuth=True)` to auth-required tools.

### 3.2 Load Testing

**Not performed in production** — would require isolated environment to avoid service disruption.

**Documented capacity based on code analysis:**
- Per-key burst limit: 10 req/s (free), 100 req/s (starter), 1000 req/s (enterprise)
- Per-minute limit: 60 req/min (free), 600 req/min (starter/pro)
- DB connections: New connection per query (no pooling) — bottleneck under load
- Redis: Used for A2A task state, no direct rate limiting

**SEC-06 (MEDIUM): No connection pooling**
`database.py` creates a new psycopg connection per query via `_get_conn()`. Under load, this will exhaust PostgreSQL connection limits. Recommend: switch to `psycopg_pool.ConnectionPool` with max_size=20.
