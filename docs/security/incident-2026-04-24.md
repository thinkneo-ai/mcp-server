## Root Cause

Functional and unit tests used **real production credentials as test fixtures** instead of synthetic/fake values. This is the author's (Claude Code agent) responsibility:

1. `test_log_redaction.py` — used real Resend production key to test the redaction engine. Should have used `re_FAKE_TEST_KEY_NOT_REAL_00000000000`.
2. `test_every_mcp_tool.py`, `test_every_a2a_skill.py`, `test_a2a_governance_e2e.py` — hardcoded the production master API key to make live calls against `mcp.thinkneo.ai`. Should have used `os.environ["THINKNEO_TEST_API_KEY"]`.
3. `src/database.py` — had `mcp_thinkneo_2026` as fallback default in `os.getenv()`. Should have defaulted to empty string.

**No customer data was exposed.** The Resend key could only send emails from `thinkneo.ai` domain. The MCP master key could call gateway tools but not access external systems. The DB password requires network access to the Docker bridge.

---

## Mitigation

1. **Resend key:** Rotated in Resend dashboard. Old key invalidated immediately.
2. **MCP master key:** Rotated in production `.env`. Container restarted. Health verified.
3. **DB password:** Default removed from source. Production uses env var.
4. **Code cleanup:** All 3 test files now use `os.environ["THINKNEO_TEST_API_KEY"]` — no hardcoded keys.
5. **sdk-typescript:** `node_modules/` removed from git, added to `.gitignore`.

---

## Lessons Learned

1. **NEVER use real credentials as test fixtures.** Use synthetic values: `re_FAKE_TEST_KEY_NOT_REAL_00000000000`, `fake_master_key_for_tests_only`.
2. **Live functional tests must read keys from environment**, not hardcode them.
3. **os.getenv() should default to empty string**, not a real password.
4. **Pre-commit hooks** should catch secrets before they reach the repo.
5. **GitGuardian caught it** — monitoring works, but prevention is better.
